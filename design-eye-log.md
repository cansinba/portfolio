## Week 1 — Seeing & deconstruction (Jul 7–13)

- 2026-07-07 | Teardown | oriorai.com | Fluid `clamp()` headings + fixed 16px body is an intentional pattern (readable body, expressive headings); section padding steps 144→112px on an 8pt scale. Stole: left-side scroll indicator.
- 2026-07-08 | Copy-work | linear.app/pricing (Basic card) · [Figma](https://www.figma.com/design/ylwSBXC2hKTnyBPI2NkgoP/) | Rebuilt the pricing card from memory. Corrections found on inspect/overlay: (1) The separators I read as `<hr>`/line elements are actually the billing_frequency element's own **border-top / border-bottom** — the divide is a neighbor's border, not a separate line element; stop hunting for line elements, check borders. (2) Hierarchy is measured, not assumed: plan name 24px, price 16px (Basic is a low tier, so price is deliberately de-emphasized) — trust the console over the "price should be biggest" heuristic. Got right: feature rows 39px tall on 55px pitch = 16px gap, matching Linear's `ul { gap:16px }`.
  - Figma craft note: inside an auto-layout, children should be **frames, not groups** — a raw Group behaves unpredictably (no own sizing, resizes wrong). Multiple elements that belong together → wrap in a frame with its own auto-layout.
- 2026-07-09 | Teardown | stripe.com (nav + hero) · [Figma](https://www.figma.com/design/ylwSBXC2hKTnyBPI2NkgoP/) `W1D3_teardown_stripe.com` | Measured: nav link 14px, hero title 48px (shrinks/grows = fluid `clamp()`, now recognized in the wild), side padding ~16px (mobile gutter), section-to-section padding 80 or 96 = 8×10 / 8×12 → **8pt scale confirmed across 3 sites, it's an industry convention, not a coincidence**. Key lesson — **"ignored" in the box model**: a value greyed as "ignored" means it's invalid in this layout context — either the element is `display:inline` (vertical margins are ignored) or it's a flex/grid item and the spacing lives in the **parent's `gap`**, not the element. So when you "can't find" the spacing, select the parent — kin to border≠line and Linear's spacer divs: spacing often lives somewhere other than the element itself. On subheadings I couldn't spot a scale — marketing pages use many sizes, but there's usually a modular scale underneath (~1.2–1.25× per step); measure the ratio to find it. Steal: (1) hero reads first / clear visual entry point; (2) the deliberately small 14px nav — shrinking the unimportant to let the hero dominate is a hierarchy technique. Would change: the colorful spiral band crosses the hero's first text line and hurts legibility — **decoration should never cost readability (figure-ground)**; fix by putting the band behind at low opacity, offsetting it off the text, or adding a scrim. Nuance: Stripe is deliberate craft, so it may be an intentional call / viewport-specific — critique is about tradeoffs, not good/bad.
